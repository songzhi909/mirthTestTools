# Mirth 消息追踪工具 - 功能需求文档

## 项目概述

为 HIS、检查系统、PACS 等医疗信息系统间的消息推送提供可视化追踪与调试能力，替代手动 SQL 查询，提升排障效率。

---

## FR-001 选择消息类型

**优先级**: P0

用户首先选择消息类型（如检查申请、检验申请、处置申请），系统根据该类型加载对应的 XML 字段定义和流程配置。

| 项目 | 说明 |
|------|------|
| 输入 | 消息类型下拉选择 |
| 输出 | 动态加载查询条件面板和流程面板 |
| 配置 | `message_types.json` 中定义每种类型对应的 XML 字段和通道流程 |

---

## FR-002 动态查询条件

**优先级**: P0

根据所选消息类型的 XML 结构，动态展示查询条件输入项。不同消息类型对应不同的业务字段。

**示例 - 检查申请新增（TJ001）：**

| 查询条件 | 来源 | 说明 |
|----------|------|------|
| EXAM_NO | `<EXAM_NO>20058752</EXAM_NO>` | 检查申请号，必填 |
| 时间范围 | Mirth 消息接收时间 | 下拉选择：最近1h/6h/24h/3天/7天 |

**示例 - 检验申请（JY001）：**

| 查询条件 | 来源 | 说明 |
|----------|------|------|
| ORDER_NO | `<ORDER_NO>xxx</ORDER_NO>` | 检验申请号，必填 |
| 时间范围 | Mirth 消息接收时间 | 下拉选择 |

**配置方式**：在 `message_types.json` 中为每种类型定义 `query_fields`，指定 XML 标签名和显示名称。

---

## FR-003 流程面板展示

**优先级**: P0

查询后按业务流程顺序展示各环节的输入/输出面板。每个流程节点显示为一个可点击的面板卡片。

**布局结构（以检查申请新增为例）：**

```
┌─────────────────────────────────────────────────────────────────┐
│  消息类型: [检查申请新增 ▼]   EXAM_NO: [________]  时间: [24h ▼]  [查询] │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐       │
│  │  HIS消息推送  │───→│ TJ001_V3生成 │───→│  V3推送PACS  │       │
│  │              │    │              │    │              │       │
│  │  输入: 3条   │    │  输入: 3条   │    │  输入: 2条   │       │
│  │  输出 ✓      │    │  输出 ✓      │    │  输出 ✓      │       │
│  │  状态: 成功   │    │  状态: 成功   │    │  状态: 失败   │       │
│  └──────────────┘    └──────────────┘    └──────────────┘       │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│  [详情面板区域 - 点击上方面板后显示]                               │
└─────────────────────────────────────────────────────────────────┘
```

**每个流程面板显示：**
- 通道名称
- 输入状态（有/无数据，用图标或颜色标识）
- 输出状态（有/无数据）
- 整体状态（成功/失败/无数据）

**流程节点配置**：在 `message_types.json` 的 `channels` 数组中按顺序定义，系统按数组顺序渲染流程图。

## FR-004 面板详情查看

**优先级**: P0

点击任意流程面板，在下方详情区域展示该环节的完整信息，包含多个 Tab 页。

| Tab 页 | 内容 | 数据来源 |
|--------|------|----------|
| 字段解析 | XML 关键字段提取 + 多消息列表 | `content_type = 1` + `extract_fields` |
| 输入XML | 上游发送到本环节的原始报文 | `content_type = 1` |
| 输出XML | 本环节发送给下游的报文 | `content_type = 1`（下一环节的输入）或响应 |
| 变量 | Mirth Channel 变量 | `content_type = 10` |
| 响应 | 下游系统返回的响应 | `content_type = 10/11` 中的响应变量 |
| 错误 | 错误堆栈信息 | `content_type = 9` |
| HIS预检 | HIS 数据库预检结果 | Oracle HIS 数据库查询 |

**多消息列表**：当同一通道查到多条消息时，"字段解析" tab 顶部显示消息列表（含接收时间和消息ID），点击可切换查看各条消息的详细信息。输出XML、变量、响应按需懒加载。

**详情区域布局：**

```
┌─────────────────────────────────────────────────────────────────┐
│  [字段解析] [输入XML] [输出XML] [变量] [响应] [错误] [HIS预检]     │
├─────────────────────────────────────────────────────────────────┤
│  共 3 条消息，点击切换:                                           │
│  ─────────────────────────────────────────────────────           │
│  ▶ [1] 2026-05-17 10:30:15  ID=12345                            │
│    [2] 2026-05-17 09:15:22  ID=12300                            │
│    [3] 2026-05-16 14:20:08  ID=12200                            │
│  ─────────────────────────────────────────────────────           │
│                                                                 │
│  事件类型: 检查申请新增                                           │
│  检查申请号: 20058752                                            │
│  检查类别: DR                                                    │
│  ...                                                            │
└─────────────────────────────────────────────────────────────────┘
```

## FR-005 消息状态标识

**优先级**: P1

流程面板通过颜色或图标直观展示状态：

| 状态 | 标识 | 含义 |
|------|------|------|
| 成功 | 绿色 / ✓ | 消息已成功发送并被接收 |
| 错误 | 红色 / ✗ | 处理失败，可点击查看错误详情 |
| 无数据 | 灰色 / - | 该环节未查到消息记录 |
| 进行中 | 黄色 / ⟳ | 已接收但未完成处理 |

## FR-006 消息类型配置化

**优先级**: P0

支持通过配置文件扩展新的消息类型，无需修改代码。

**配置文件**: `message_types.json`

**配置结构：**

```json
{
  "消息类型名称": {
    "code": "业务代码",
    "query_fields": [
      {
        "label": "显示名称",
        "xml_tag": "XML标签名",
        "required": true
      }
    ],
    "channels": [
      {
        "name": "环节名称",
        "id": "Channel ID",
        "table_suffix": "表后缀",
        "response_var": "响应变量名",
        "response_content_type": 10
      }
    ],
    "his_precheck": {
      "description": "预检描述",
      "keyword_field": "EXAM_NO",
      "sql": "Oracle SQL，{keyword} 为占位符",
      "status_field": "status",
      "fail_message_field": "status_name",
      "fields": [
        {"key": "字段名", "label": "显示名称"}
      ]
    }
  }
}
```

**字段说明：**

| 字段 | 必填 | 说明 |
|------|------|------|
| `code` | 是 | 业务代码（如 TJ001） |
| `query_fields` | 是 | 查询条件列表，从 XML 中提取 |
| `query_fields[].label` | 是 | 界面显示名称 |
| `query_fields[].xml_tag` | 是 | XML 中的标签名，用于 LIKE 查询 |
| `query_fields[].required` | 否 | 是否必填，默认 false |
| `channels` | 是 | 流程节点数组，按业务顺序排列 |
| `channels[].name` | 是 | 环节显示名称 |
| `channels[].id` | 是 | Mirth Channel ID |
| `channels[].table_suffix` | 是 | 数据库表后缀（通常与 id 相同） |
| `channels[].response_var` | 否 | 响应报文中的变量名 |
| `channels[].response_content_type` | 否 | 响应 content_type（10 或 11） |
| `his_precheck` | 否 | HIS 数据库预检配置，见 FR-009 |
| `his_precheck.keyword_field` | 是 | 取哪个 query_fields 的值作为 SQL 的 `{keyword}` |
| `his_precheck.sql` | 是 | Oracle SQL 模板，`{keyword}` 为占位符 |
| `his_precheck.status_field` | 是 | 结果中表示状态的字段名（"0"=失败，"1"=成功） |
| `his_precheck.fail_message_field` | 是 | 失败时取该字段值作为弹窗提示 |
| `his_precheck.fields` | 是 | 结果展示的列映射 |

## FR-007 数据库连接配置

**优先级**: P0

支持通过配置文件修改数据库连接信息，无需重新打包。

**配置文件**: `config.ini`

```ini
[database]
host = 127.0.0.1
port = 3306
user = root
password = xxx
database = mirth_data

[oracle]
host = 192.168.x.x
port = 1521
service_name = orcl
user = his_user
password = his_password
```

| 配置段 | 说明 |
|--------|------|
| `[database]` | Mirth 消息数据库（MySQL），用于查询 Mirth 通道消息 |
| `[oracle]` | HIS 数据库（Oracle），用于 HIS 预检查询。未配置时跳过预检 |

**优先级**: P1

数据库查询在后台线程执行，不阻塞 GUI 界面。查询过程中显示加载状态。

## FR-009 HIS 数据库预检

**优先级**: P0

查询 Mirth 消息前，先连接 Oracle HIS 数据库校验业务数据状态。预检不通过时弹窗提示并中止后续查询，避免在 HIS 数据异常时做无效排查。

**查询流程：**

```
用户输入检查申请号 → 点击查询
  → HIS预检 (Oracle)
     ├── 未查到记录 → 弹窗"HIS中未查到该申请单数据" → 中止
     ├── status=0   → 弹窗显示失败原因（如"门诊未缴费"）→ 中止
     ├── 连接/SQL错误 → 弹窗显示错误 → 中止
     └── status=1   → 继续 Mirth 通道查询
```

**配置方式**：在 `message_types.json` 的消息类型中添加 `his_precheck` 字段，详见 FR-006。

**预检结果展示**：详情区域的 "HIS预检" tab 显示查询结果（就诊类型、检查类别、状态、收据号、校对时间等）。

**技术要点**：
- 使用 `python-oracledb` thick 模式（兼容 Oracle 11g）
- Oracle Instant Client DLLs 已内置到 exe，目标机器无需安装
- HIS 预检结果在详情区域的 "HIS预检" tab 中持久展示，切换通道不影响

## FR-010 多消息列表与选择

**优先级**: P0

同一申请单在同一通道中可能产生多条 Mirth 消息（如重发、重推）。支持列出所有消息并选择查看。

**行为说明：**

| 场景 | 行为 |
|------|------|
| 0 条消息 | 面板显示"输入: ✗"，详情显示"(无输入报文)" |
| 1 条消息 | 面板显示"输入: 1条"，直接展示详情 |
| 多条消息 | 面板显示"输入: N条"，"字段解析" tab 顶部显示可点击消息列表 |

**消息列表格式：**

```
共 3 条消息，点击切换:
─────────────────────────────────────────────────────────
▶ [1] 2026-05-17 10:30:15  ID=12345
  [2] 2026-05-17 09:15:22  ID=12300
  [3] 2026-05-16 14:20:08  ID=12200
─────────────────────────────────────────────────────────
```

- `▶` 标记当前选中的消息
- 点击任意行切换到该消息的详情
- 输入XML 立即显示，输出XML/变量/响应 懒加载（后台线程查询）

## FR-011 消息类别查询条件

**优先级**: P1

工具栏新增"类别"输入框，用户可输入消息类别值。该值通过 `{category}` 变量传入 SQL 模板，用于在 WHERE 条件中过滤消息类别。

**使用方式**：在 `sql_templates.json` 的 SQL 中添加 `{category}` 占位符。不添加则该变量被忽略，不影响现有查询。

---

## FR-012 查询进度指示

**优先级**: P1

查询过程中在状态栏实时显示当前执行步骤，方便用户了解查询进度。

**显示格式**：`[当前步/总步数] 步骤描述`

**示例**：
```
[1/4] HIS预检查询中...
[2/4] 查询通道: HIS消息推送...
[3/4] 查询通道: TJ001_V3生成...
[4/4] 查询通道: V3推送PACS...
```

切换查看消息详情时显示 `加载消息详情...`。
